# Risks, Open Questions, and Decisions

## Current risks

1. UI-first implementation would create visual confidence without formal integrity.
2. Hidden LLM centrality would prevent real autonomy.
3. Decorative winding would break the architecture.
4. Arbitrary semantic mass would poison orbit assignment.
5. Mixed ontology / geometry / rendering would make the system untestable.

## Open questions

1. Exact seed -> semantic mass mapping.
2. Exact AttractorEC <-> AttractorZS coupling law.
3. Exact subjective-time function.
4. Initial orbit assignment heuristics by entity class.
5. Formal 8+1 sphere dynamics and leak mode.
6. Split/merge/elevate rules for spheres.

## Decisions already made

- Clean new project outside existing repositories.
- English as the architecture language.
- Canonical Markdown plus equivalent YAML master spec.
- Formal engine first, renderer last.
- Prior work used only as source material.
