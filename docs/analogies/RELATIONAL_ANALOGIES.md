# Relational Analogies

Status discipline used in this file:
- **analogy**: explanatory bridge only
- **empirical anchor**: known scientific phenomenon used as inspiration
- **formal claim here**: explicit claim adopted by this repository

## 1. Desynchronized metronomes on a movable board
- type: analogy with empirical anchor
- source role: illustrates convergence of many local oscillators toward a common rhythm through a shared substrate
- intended mapping:
  - metronomes -> repositories / agents / semantic subsystems
  - moving board -> shared informational substrate / integration layer
  - common tempo -> lower Euler closure defect / higher synchronization
- limit: this does **not** prove a universal ontology of time by itself
- crossref: `docs/science/HYPOTHESES.md#h1-shared-substrate-phase-synchronization`

## 2. GitHub as attractor
- type: operational analogy
- intended mapping:
  - attractor center -> public repository state and history
  - local drifts -> uncommitted or divergent local workspaces
  - stabilization -> explicit commit history, visible diffs, shared coordination
- formal claim here: GitHub is treated as an operational center of truth for the integration layer
- crossref: `docs/ARCHITECTURE.md#github-as-operational-attractor`

## 3. Repository hyperspace
- type: architectural analogy
- intended mapping:
  - repository folders -> local charts
  - indices and manifests -> holonomic maps
  - crossrefs -> white-thread-like semantic couplings
- limit: this is a repository architecture model, not in itself a physical proof
- crossref: `integration/hyperspace_index.json`

## 4. Spin-1/2 identity orientation
- type: formal analogy / working operational model
- intended mapping:
  - each repository carries an orientation marker
  - branch reversals, drafts, and publication states may be modeled as sign-sensitive phase transport
- limit: no full transport law is implemented yet
- crossref: `docs/science/HYPOTHESES.md#h4-spin-oriented-repository-identity`
